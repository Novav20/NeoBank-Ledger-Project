# Neobank Ledger API: B2B FinTech Backend

## Project Vision
A robust, highly transactional, and auditable Ledger API built with C# 14 / .NET 10. Designed for B2B FinTech operations where consistency and performance are non-negotiable.

---

## Project Structure (Docs-as-Code)

### /docs
The Source of Truth for the entire system.
- **00_meta/**: Project governance and AI orchestration.
    - **adr/**: Architecture Decision Records. Every major technical choice is recorded here.
    - **orchestration/**: The Shared Memory for AI agents.
        - templates/: Formal structures for plans, Copilot prompts, and responses.
        - prompts/: Active instructions sent to GitHub Copilot.
        - responses/: Data and code proposals generated by Copilot.
        - logs/: session-state.md tracks the project's current status and AI partnership state.
    - **plans/**: Weekly execution plans (e.g., W11.md, W12.md).
    - **Traceability.md**: High-level timeline and daily progress logs.
- **02_analysis/**: Business Requirements, Non-Functional Requirements (NFRs), and Domain Modeling.
- **03_architecture/**: Database Schema (Mermaid), API Contracts, and System Design.

### /src
The implementation layer. Clean Architecture following C# 14 / .NET 10 standards.
- **Domain**: Core entities and business logic.
- **Application**: Use cases and orchestration.
- **Infrastructure**: Data access (EF Core 10), SQL Server integration, and external services.
- **API**: REST controllers and middleware.

### /tests
Automated verification suite.
- **Unit**: Business logic and domain tests.
- **Integration**: API and Database layer verification.

### Root Manifests
- **GEMINI.md**: The system manifest for the Local Architect (Gemini CLI).
- **.github/copilot-instructions.md**: Constraints and role definitions for the Execution Engine (GitHub Copilot).

---

## AI Orchestration Workflow
We employ a Senior-Junior collaboration model:
1. **Local Architect (Gemini CLI)**: Analyzes requirements, drafts ADRs, and generates structured instructions in docs/00_meta/orchestration/prompts/.
2. **Execution Engine (GitHub Copilot)**: Receives prompts, generates high-speed code/research, and outputs to docs/00_meta/orchestration/responses/.
3. **Verification**: Gemini CLI validates the output against the ADRs before finalizing the implementation.
