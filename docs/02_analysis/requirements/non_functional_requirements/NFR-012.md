# NFR-012: Maintain an Open-Source Dependency SLA with CVE Patching Discipline

## Statement
The Ledger system shall maintain a service-level policy for open-source dependencies that defines vulnerability response and CVE patching discipline.

## Rationale
[[BPA_Report#4.2 Consolidated NFR Registry|BPA NFR-RISK-01]] defines the risk target as an OSS dependency SLA plus CVE patching policy. [[chuen_2017]] warns that grassroots open-source operational risk can leave vulnerabilities unpatched when governance or funding is missing, and [[mishra_2025]] reinforces the need for supply-chain provenance and code transparency. The resulting requirement is a direct backlog expression of the BPA risk row.

## Acceptance Criteria
- [ ] The project defines a response policy for open-source dependency vulnerabilities.
- [ ] The project defines how CVE severity or urgency maps to patching priority.
- [ ] Dependency risk can be reviewed through periodic audit or compliance checks.

## Notes
This requirement covers dependency governance and patch response discipline; it does not prescribe a specific patching implementation.
