# NFR-010: Use RTGS Settlement Without Net-Settlement Exposure

## Statement
The Ledger system shall settle pre-funded retail payments in real time using a real-time gross settlement model that does not create net-settlement exposure.

## Rationale
[[BPA_Report#4.2 Consolidated NFR Registry|BPA NFR-COMP-04]] defines the compliance target as an RTGS settlement model with no net-settlement exposure. [[sonnino_2021]] states that FastPay is designed as an RTGS system that eliminates credit and counter-party risks of net settlement by settling pre-funded retail payments in real time. This makes the settlement model directly traceable and testable.

## Acceptance Criteria
- [ ] Each eligible payment is settled individually rather than being deferred into a netted batch.
- [ ] The system does not introduce credit or counter-party exposure from net settlement.
- [ ] Settlement completion occurs in real time for pre-funded retail payments.

## Notes
This requirement covers settlement structure and exposure elimination, not the internal consensus mechanism.
