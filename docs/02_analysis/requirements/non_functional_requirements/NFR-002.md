# NFR-002: Maintain Intra-Cluster Latency Below Normal-Load Threshold

## Statement
The Ledger system shall keep intra-cluster latency below 210 milliseconds under normal load.

## Rationale
[[BPA_Report#4.2 Consolidated NFR Registry|BPA NFR-PERF-01]] defines the latency target as <210ms intra-cluster latency under normal load. [[al_bassam_2018]] reports client-perceived latency in the 69ms to 210ms range and provides a direct performance baseline for the sharded BFT design. This makes the performance threshold directly traceable without introducing any open gap.

## Acceptance Criteria
- [ ] Under normal load, measured intra-cluster latency remains below 210 milliseconds.
- [ ] The system can be benchmarked against the reported normal-load latency envelope without exceeding the stated ceiling.
- [ ] The latency target is observable in performance-test results for the ledger core pathways.

## Notes
This NFR stays close to the BPA statement so the threshold remains explicit and testable. The cited research note supports the performance envelope directly, so no gap entry is required for this item.
