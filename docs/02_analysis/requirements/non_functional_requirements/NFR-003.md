# NFR-003: Sustain 2,500 TPS in a 7-Node LAN Cluster

## Statement
The Ledger system shall sustain at least 2,500 transactions per second in a 7-node LAN deployment.

## Rationale
[[BPA_Report#4.2 Consolidated NFR Registry|BPA NFR-PERF-02]] sets the performance target at ≥2,500 TPS in a 7-node LAN. [[barger_2021]] reports the same throughput figure in a 7-node LAN setup and provides a direct benchmark for the BFT ordering layer. This makes the requirement directly traceable without introducing any open gap.

## Acceptance Criteria
- [ ] In a 7-node LAN deployment, measured throughput reaches at least 2,500 TPS.
- [ ] The benchmark scenario uses the same cluster scale and workload class described in the source note.
- [ ] The throughput result is reproducible through a documented load test.

## Notes
This NFR keeps the BPA threshold intact so the performance target remains measurable and easy to verify.
