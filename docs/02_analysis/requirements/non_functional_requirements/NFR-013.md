# NFR-013: Use LevelDB by Default and Justify Private Data Collections

## Statement
The Ledger system shall use LevelDB as the default ledger database and shall enable private data collections only when a documented use case justifies them.

## Rationale
[[BPA_Report#4.2 Consolidated NFR Registry|BPA NFR-CONFIG-01]] defines the configuration target as LevelDB over CouchDB with private data collections only if justified. [[guggenberger_2022]] reports that LevelDB improves write throughput by a factor of three, while private data collections reduce throughput by 2x to 3x. This makes the configuration choice directly traceable and operationally testable.

## Acceptance Criteria
- [ ] The default ledger database is LevelDB.
- [ ] CouchDB is not selected unless there is a documented justification for doing so.
- [ ] Private data collections are enabled only when their use is explicitly justified.

## Notes
This requirement captures the default storage configuration and the exception rule for private data collections.
