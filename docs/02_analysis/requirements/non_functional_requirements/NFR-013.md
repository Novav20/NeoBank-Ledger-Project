# NFR-013: Use RocksDB by Default and Justify Private Data Collections

## Statement
The Ledger system shall use RocksDB as the default ledger database and shall enable private data collections only when a documented use case justifies them.

## Rationale
[[BPA_Report#4.2 Consolidated NFR Registry|BPA NFR-CONFIG-01]] defines the configuration target as a high-performance embedded ledger store with private data collections only if justified. RocksDB is the active implementation choice because its multi-threaded compaction and modern `Span`/`Memory`-friendly access patterns fit the .NET persistence layer better, while preserving the same exception-based rule for private data collections.

## Acceptance Criteria
- [ ] The default ledger database is RocksDB.
- [ ] CouchDB is not selected unless there is a documented justification for doing so.
- [ ] Private data collections are enabled only when their use is explicitly justified.

## Notes
This requirement captures the default storage configuration and the exception rule for private data collections. RocksDB is the current choice because it provides better write-path compaction behavior for the .NET persistence layer without changing the traceability contract.
