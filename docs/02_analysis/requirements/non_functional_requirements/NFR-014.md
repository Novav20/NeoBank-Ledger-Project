# NFR-014: Keep the m-Node Shard Ratio Between 20% and 25%

## Statement
The Ledger system shall configure the m-node shard ratio between 20% and 25%.

## Rationale
[[BPA_Report#4.2 Consolidated NFR Registry|BPA NFR-CONFIG-02]] defines the configuration target as a 20% to 25% m-node shard ratio. [[liu_2025]] reports that optimal throughput is reached when the m-node proportion sits between 20% and 25%, making the ratio a direct performance and configuration target rather than an inferred one.

## Acceptance Criteria
- [ ] The configured m-node proportion stays within the 20% to 25% range.
- [ ] The shard plan can be reviewed to confirm the ratio before deployment.
- [ ] Throughput benchmarks use the same shard-ratio target described in the requirement.

## Notes
This requirement captures the shard composition target only; it does not redefine the underlying consensus protocol.
