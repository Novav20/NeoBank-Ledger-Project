# NFR-009: Enforce Fair Ordering for Order-Sensitive Transactions

## Statement
The Ledger system shall apply a fair-ordering policy to order-sensitive transactions in the Ordering Service.

## Rationale
[[BPA_Report#4.2 Consolidated NFR Registry|BPA NFR-COMP-03]] defines the compliance target as a fair-ordering policy on the Ordering Service. [[li_2024]] describes fair consensus as a modular framework for FIFO, random, or blind ordering, with ordering metadata and independent re-execution used to verify results and mitigate MEV. This makes the fairness requirement directly traceable without introducing a gap.

## Acceptance Criteria
- [ ] Order-sensitive transactions are processed according to a documented fair-ordering policy.
- [ ] The Ordering Service emits enough ordering metadata for independent verification.
- [ ] The selected policy prevents MEV-relevant reordering of order-sensitive transactions.

## Notes
This requirement covers ordering fairness only; it does not define the broader consensus algorithm.
