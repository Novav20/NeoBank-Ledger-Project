# NFR-017: Distinguish Committed and Audited Finality

## Statement
The Ledger system shall distinguish committed finality from audited finality in its ledger state and audit outputs.

## Rationale
[[BPA_Report#4.2 Consolidated NFR Registry|BPA NFR-SAFE-02]] defines the requirement as committed vs. audited finality formally distinguished. [[mishra_2025]] describes committed transactions and audited transactions as separate notions, with asynchronous background auditing and receipts providing the distinction. This makes the requirement directly traceable without introducing a gap.

## Acceptance Criteria
- [ ] The system exposes whether a transaction is committed or audited.
- [ ] Audit reconciliation can be performed without changing the committed history.
- [ ] Receipts or equivalent evidence allow the audited state to be verified independently.

## Notes
This requirement covers finality semantics only; it does not define audit proof delivery.
