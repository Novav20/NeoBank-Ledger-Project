# NFR-016: Maintain an Intrusion-Tolerant Inter-Ledger Overlay with Multi-Path Routing

## Statement
The Ledger system shall route inter-ledger traffic through a multi-path intrusion-tolerant overlay that preserves progress during path outages, node crashes, and network partitions.

## Rationale
[[BPA_Report#4.2 Consolidated NFR Registry|BPA NFR-INFRA-01]] defines the infrastructure target as a multi-path intrusion-tolerant inter-ledger overlay. [[trestioreanu_2021]] describes SPON as an intrusion-tolerant overlay that uses multiple disjoint paths, flooding, and relay redundancy to route around BGP hijacking, node crashes, and partitions. This makes the transport-resilience requirement directly supported by the evidence.

## Acceptance Criteria
- [ ] Inter-ledger traffic can be routed across multiple disjoint paths.
- [ ] The overlay continues to make progress when a primary path is unavailable.
- [ ] The overlay remains functional under BGP hijacking or network-partition scenarios when at least one non-adversarial path exists.

## Notes
This requirement covers inter-ledger transport resilience only; it does not change the consensus or settlement protocol.
