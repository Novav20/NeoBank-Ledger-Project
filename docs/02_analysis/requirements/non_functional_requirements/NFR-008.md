# NFR-008: Tag Trade Events with LEI, UTI, and 1ms Timestamps

## Statement
The Ledger system shall record each trade event with a Legal Entity Identifier, a Unique Transaction Identifier, and a timestamp precision of 1 millisecond.

## Rationale
[[BPA_Report#4.2 Consolidated NFR Registry|BPA NFR-COMP-02]] defines the compliance target as LEI, UTI, and 1ms timestamps per trade event. [[chuen_2017]] explicitly identifies the MiFID II requirement for LEI, UTI, and timestamping at 1ms precision. This makes the audit trail requirement directly traceable to the BPA and the cited handbook.

## Acceptance Criteria
- [ ] Every trade event is recorded with an LEI and a UTI.
- [ ] Trade-event timestamps are stored with 1ms precision.
- [ ] The trade-event record is suitable for regulatory review and audit traceability.

## Notes
This requirement covers event metadata for regulatory traceability, not the broader audit-proof delivery model.
