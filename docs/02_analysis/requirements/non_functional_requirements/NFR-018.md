# NFR-018: Formally Verify Fork-Freedom with IVy and Z3

## Statement
The Ledger system shall be formally verified for fork-freedom using IVy, Z3, or an equivalent formal verification toolchain.

## Rationale
[[BPA_Report#4.2 Consolidated NFR Registry|BPA NFR-SAFE-03]] defines the safety target as formal verification of fork-freedom. [[praveen_2024]] reports an IVy and Z3 proof of fork-freedom and safety-error-freedom using inductive invariants and monitors. This makes the formal-verification requirement directly traceable without a new gap.

## Acceptance Criteria
- [ ] A formal model of the consensus protocol exists.
- [ ] The model is checked with IVy, Z3, or an equivalent formal verification toolchain.
- [ ] The proof artifacts demonstrate fork-freedom and safety-error-freedom.

## Notes
This requirement captures proof obligation and verification tooling, not runtime safety monitoring.
