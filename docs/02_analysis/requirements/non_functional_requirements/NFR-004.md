# NFR-004: Configure the Consensus Timer Using a 3σ Safety Margin

## Statement
The Ledger system shall configure the initial consensus timer $\tau_0$ to be at least three standard deviations above the expected consensus time.

## Rationale
[[BPA_Report#4.2 Consolidated NFR Registry|BPA NFR-PERF-03]] defines the timer target as $\tau_0 \ge 3\sigma$. [[chan_2018]] recommends setting the initial timer value with a three-standard-deviation margin to accommodate variance in consensus processing. The requirement is therefore directly supported by the BPA row and the cited source note.

## Acceptance Criteria
- [ ] The configured initial consensus timer is at least $3\sigma$ above the expected consensus-time baseline.
- [ ] The timer configuration can be reproduced from the documented consensus timing inputs.
- [ ] Consensus timing simulations confirm that the chosen timer satisfies the stated safety margin.

## Notes
This NFR captures timer tuning rather than a runtime performance limit. It is intentionally phrased as a configuration requirement so the threshold can be validated before deployment.
