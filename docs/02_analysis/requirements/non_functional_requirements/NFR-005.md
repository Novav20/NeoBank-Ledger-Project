# NFR-005: Maintain Tiered Latency SLA for Intra-Continental and WAN Traffic

## Statement
The Ledger system shall keep intra-continental confirmation latency below 100 milliseconds and shall keep WAN latency within a 1.5-second ceiling.

## Rationale
[[BPA_Report#4.2 Consolidated NFR Registry|BPA NFR-PERF-04]] combines the <100ms target and 1.5s WAN ceiling into a tiered SLA. [[sonnino_2021]] reports sub-100ms intra-continental confirmation latency for FastPay and about 200ms inter-continental latency, while [[guggenberger_2022]] reports around 1.2s intercontinental latency for public transactions and greater than 1.7s for private transactions. The BPA threshold sits inside that WAN envelope and preserves an explicit tiered SLA.

## Acceptance Criteria
- [ ] Intra-continental confirmation latency remains below 100 milliseconds under the target load profile.
- [ ] WAN latency remains within the 1.5-second ceiling for the tiered SLA profile.
- [ ] The latency target can be reproduced in a documented performance benchmark.

## Notes
The WAN ceiling is a BPA synthesis of the cited intercontinental measurements, not a separate gap.
