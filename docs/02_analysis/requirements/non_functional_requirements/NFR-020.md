# NFR-020: Target 7 PBFT Shards for a 1,000-Node Deployment

## Statement
The Ledger system shall target a PBFT shard count of 7 for a 1,000-node deployment.

## Rationale
[[BPA_Report#4.2 Consolidated NFR Registry|BPA NFR-CONFIG-03]] defines the configuration target as PBFT k=7 shards at 1,000 nodes. [[wang_2026]] reports that the optimal shard count for a 1,000-node system using PBFT and Atomix is 7, so the target is supported directly by the source evidence.

## Acceptance Criteria
- [ ] The deployment plan uses 7 PBFT shards for a 1,000-node target size.
- [ ] The shard plan can be reviewed before deployment to confirm the target count.
- [ ] Benchmark documentation references the same shard count used in the configuration target.

## Notes
This requirement captures the sizing target for the deployment plan; it does not prescribe a different consensus family.
