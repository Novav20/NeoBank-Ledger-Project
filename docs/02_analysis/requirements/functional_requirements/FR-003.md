# FR-003: Enforce Double-Entry Integrity

## Statement
Every transaction MUST satisfy the dual-entry axiom where Σ(Debits) - Σ(Credits) = 0.

## Rationale
This requirement is grounded in [[BPA_Report#4.1 Foundational Principles (Core Business Rules)|BPA Section 4.1]], which defines double-entry balance as a non-negotiable core business rule for the ledger.

## Acceptance Criteria
- [ ] Transactions that do not balance are rejected.
- [ ] Every accepted transaction preserves Σ(Debits) - Σ(Credits) = 0.
- [ ] The requirement can be verified through automated ledger balance checks.

## Notes
This requirement is intentionally atomic and excludes normalization, ordering, and authorization concerns.
