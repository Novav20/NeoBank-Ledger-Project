# FR-004: Preserve Integer Precision for Financial Quantities

## Statement
All financial quantities MUST be handled as integers in the smallest currency unit to prevent floating-point errors.

## Rationale
This requirement is grounded in [[BPA_Report#4.1 Foundational Principles (Core Business Rules)|BPA Section 4.1]], which requires integer arithmetic for financial quantities to prevent fractional data loss and rounding errors.

## Acceptance Criteria
- [ ] Financial amounts are represented using integer arithmetic.
- [ ] No floating-point storage or calculation is used for monetary values.
- [ ] The requirement can be verified by inspecting persisted amounts and arithmetic paths.

## Notes
This requirement applies to all monetary values handled by the ledger, regardless of transaction type.
