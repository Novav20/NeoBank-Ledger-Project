# FR-001: Authenticate and Authorize Transaction Commands

## Statement
The Ledger API shall verify the identity and permissions of the API caller before processing any transaction command.

## Rationale
This requirement is grounded in [[BPA_Report#2. Analysis of the Current State (As-Is)|BPA access-control behavior]] that requires strict access controls to overcome the trust deficit between non-affiliated parties, and it supports the UC-01 entry point before any execution or normalization occurs.

## Acceptance Criteria
- [ ] A request from an authenticated and authorized caller is allowed to proceed.
- [ ] A request from an unauthenticated caller is rejected before any transaction processing begins.
- [ ] A request from an authenticated but unauthorized caller is rejected before any transaction processing begins.

## Notes
This FR is intentionally limited to caller identity and permission checks. Command normalization is handled separately in FR-002.
