# FR-002: Normalize Transaction Commands into Structured Events

## Statement
The Ledger API shall transform each validated transaction command into a structured internal event representation before downstream execution or ordering begins.

## Rationale
This requirement is grounded in [[BPA_Report#5.2 To-Be Flow (Event Sourcing + Shards + Fabric-like EOV)|BPA Section 5.2]], which defines Command & Intent Capture as the first step in the EOV pipeline, and it supports the project goal of aligning inbound message handling with ISO 20022 structured messaging once the detailed schema mappings are available.

## Acceptance Criteria
- [ ] A validated command is transformed into a structured internal event.
- [ ] The transformed event is produced before Execute processing begins.
- [ ] The requirement can later be mapped to an ISO 20022-compliant payload schema once the schema research is completed.

## Notes
Specific ISO 20022 message profiles are intentionally left as placeholders until the schema research is completed.
