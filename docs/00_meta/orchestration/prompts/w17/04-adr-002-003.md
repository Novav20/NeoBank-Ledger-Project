## ADR-002: Deterministic Finality via PBFT
1. **Decision**: Use **PBFT (Practical Byzantine Fault Tolerance)** for the core consensus cluster to achieve immediate deterministic finality.
2. **Rationale**: B2B financial settlement requires fork-freedom (probability of reversal = 0). RAFT (CFT-only) is insufficient for trust-sensitive partners.
3. **Consequences**:
   - Positive: Immediate finality; mature library support (BFT-SMaRt).
   - Negative: Quadratic communication overhead ($O(n^2)$).
4. **References**: Benedetti et al. (2022); Barger et al. (2021); Chan et al. (2018).

## ADR-003: Sharding Topology via MSSP (m-nodes)
1. **Decision**: Adopt a **Multi-Shard Storage Protocol (MSSP)** with an **m-node ratio of 20%-25%**.
2. **Rationale**: Eliminates the need for complex Two-Phase Commit (2PC) for cross-shard transactions by allowing m-nodes to participate in multiple consensus zones.
3. **Consequences**:
   - Positive: Linear throughput scaling; reduced confirmation latency.
   - Negative: Increased storage and bandwidth requirements for m-nodes.
4. **References**: Liu et al. (2025); Yu et al. (2023); Wang et al. (2026).
