# Weekly Plan: week 19 (May 04 - May 08)

## Context
This week absorbs the deferred Infrastructure work from Week 18. The goal is to understand and complete the RocksDB-backed persistence path in a slower, more deliberate way before moving toward the Event Store implementation.

## SMART Objectives
- **Specific:** Finish the Infrastructure persistence layer for RocksDB.
- **Measurable:** Repository and store methods compile cleanly and persist DTOs with the agreed key schema.
- **Achievable:** Reuse the Domain Layer and the DTO contracts already created.
- **Relevant:** The ledger needs a stable persistence base before event sequencing and projections.
- **Time-bound:** End the week with the persistence layer understood and implementation-ready.

### Daily Schedule

### Monday (2026-05-04) - Persistence Foundation
- [x] **Infrastructure Dependencies**: Remove LevelDB.Net and install RocksDb.Net.
- [x] **Store Engine**: Create `RocksDbStore.cs` for low-level Key-Value access.
- [x] **Static Mapping**: Implement pure C# mappers (e.g., `AccountMapper`) to avoid AutoMapper.

### Tuesday (2026-05-05) - Repository Bridge
